Release Notes

Calendar v0.2

DONE
- бизнес-логика выборки событий занесена в модель event и разнесена по функциям;
- имена переменных в модели event заменены на более понятные, н-р: is_periodical,
  periodical, day_of_week, etc.;
- для большей sql-совместимости между различными базами данных при выборке
  используется оператор LIKE;
- модель event покрыта тестами (42 шт.);
- пофикшено 4 бага, выявленных в процессе покрытия тестами;
- добавлены человекопонятные сообщения об ошибках:
    вследствие отсутствия навыков детального проектирования реализованы
    несколько убого, поскольку нет соглашения по передачи формата ответа
    от бэкенда к фронтенду;
- добавлены коды ошибок, интерпритация которых осуществляется из config/erros.yml:
    введены в экспериментальном порядке, весьма вероятно в промышленном
    производстве ПО коды ошибок реализуются и обрабатываются иначе;
- добавлена подсветка невалидных текстовых полей;
- запрос на авторизацию переведен на ajaх для выдачи сообщений в случае ошибки;
- кнопка выхода из кабинета оформлена в виде изображения;
- добавлена документация по контролерам и интерактивным GUI-элементам, а также
  mockup'ы.
    Как оказалось, разработка и проектирование ПО - не такая уж простая
    задача, как это казалось сперва, требующая соотвествующих
    навыков и знаний, но не теряющая от этого своей привлекательности:
    спроектировать и детально описать на бумаге работающую систему -
    такой навык дорогого стоит. В связи с этим весьма занимательно высказывание
    Алана Купера в книге "Психбольница в руках пациентов" относительно
    противопоставления производства ПО производству высокобюджетного фильма, где
    Алан говорит, что в отличие от ПО фильм изначально проектируется детально
    на бумаге, поскольку пересъемка сцен по каждой прихоти режиссера
    неминуемо влекла бы за собой существенное возрастание расходов.

BUGS
- не работает рассылка сообщений о восстановлении пароля c heroku;
- не обрабатываются исключения при запросе несуществующих маршрутов;
